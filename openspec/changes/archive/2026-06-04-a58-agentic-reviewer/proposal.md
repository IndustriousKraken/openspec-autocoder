## Why

The code-reviewer is the last high-value LLM step still on the single-shot HTTP path: it assembles every touched file's full contents into one prompt, sends it to a configured LLM API in one `LlmClient::complete` call, and parses a `VERDICT:` line out of the free-text response. That shape has three standing problems. (1) The full-file-dump prompt hits `reviewer.prompt_budget_chars` (default 2,000,000) on large passes and silently truncates touched files behind a `## Skipped (budget exhausted)` footer — the reviewer reviews less than the whole change. (2) The verdict is scraped from prose, and the operator-triggered re-review composer mis-defaults a malformed verdict to `Approve` (the documented bug). (3) It is Anthropic/OpenAI-HTTP-shaped, so the reviewer cannot share the CLI-wrapped, provider-swappable session the rest of the fleet is moving onto.

a56 landed the shared `agentic_run` primitive and the per-role `submit_*` submission framework; a57 made the advisory audits its first consumers. This change does the same for the reviewer: an agentic, CLI-wrapped reviewer that reads files on demand (no pre-dump, no truncation), runs in a read-only sandbox, and returns a schema-validated verdict via a `submit_review` MCP tool. A bad verdict becomes a correctable tool call; no valid submission discards the review and alerts the operator instead of approving by default — retiring the malformed-verdict bug in the agentic path.

## What Changes

**`reviewer.kind` selects the transport (code-reviewer).** A new `reviewer.kind: oneshot | agentic` field, default `oneshot`, selects between the existing HTTP one-shot path and the new agentic path. The default stays `oneshot` deliberately: the agentic path runs through a56's `ClaudeStrategy`, which only reaches Anthropic-shaped endpoints, so non-Anthropic reviewers (Qwen/Ollama via `openai_compatible`/`ollama`) cannot go agentic until the opencode strategy (a60) lands. The default flip is deferred to that point; a58 ships agentic as opt-in.

**Agentic reviewer mode (code-reviewer).** When `reviewer.kind: agentic`, the reviewer runs through a56's `agentic_run` primitive with a read-only sandbox whose allowed tools are `Read`, `Glob`, `Grep` (NO `Bash`, NO `Write`/`Edit`) plus the `submit_review` MCP tool, and `ORCH_MCP_ROLE = reviewer`. The rendered prompt carries the change briefs, the unified diff, and the list of changed file paths; full file contents are NOT pre-dumped — the agent reads whatever files it needs on demand, so `reviewer.prompt_budget_chars` does not apply and nothing is skipped for budget. The agent returns its verdict and concerns by calling `submit_review`. The path honors `reviewer.mode` (per_change vs bundled) identically, and produces the same `ReviewResult` the one-shot path produces, so per_change dispatch, `auto_revise` revision comments, the operator re-review verb, and the revision/re-review caps all keep working unchanged. `reviewer.command` (default `claude`) selects the CLI; a non-`claude` command resolves its strategy via a55/a56's `provider → CLI` rule and returns "strategy not yet implemented" until a60.

**`submit_review` MCP tool (executor).** Built on a56's per-role submission framework, the MCP child advertises `submit_review` when `ORCH_MCP_ROLE = reviewer`. Its payload is `{ verdict: Approve | Block, summary, concerns: [{ title, detail, anchor, should_request_revision, actionable_request }] }`; the schema enforces the verdict enum and requires a non-empty `actionable_request` whenever `should_request_revision` is true. A schema-invalid submission is surfaced to the agent as a correctable tool error; after the session the daemon `consume_submission`s the payload into a `ReviewResult`. A session that ends with no valid submission causes the daemon to DISCARD the review (no verdict written, NOT an implicit approval) AND post a chatops alert — this is the agentic retirement of the malformed-verdict-defaults-to-approve bug.

## Impact

- **Affected specs:**
  - `code-reviewer` — MODIFIED `AI-driven code-quality review` (scoped to the `oneshot` transport; behavior otherwise unchanged); ADDED `Agentic reviewer mode`.
  - `executor` — ADDED `submit_review MCP tool returns the reviewer verdict`.
- **Affected code:**
  - `autocoder/src/code_reviewer.rs` — `review_pr_at_state` dispatches on `reviewer.kind`; the agentic branch builds the briefs+diff+file-list prompt and runs `agentic_run` with the read-only sandbox + `submit_review`; `consume_submission` → `ReviewResult`.
  - `autocoder/src/config.rs` — `reviewer.kind` field (default `oneshot`); `reviewer.command` (default `claude`).
  - `autocoder/src/mcp_askuser_server.rs` — register `submit_review` + its schema under a56's framework, gated on `ORCH_MCP_ROLE = reviewer`.
  - `autocoder/src/revisions.rs` — the rerun composer's no-valid-submission path discards + alerts (agentic) instead of the verdict-default.
- **Operator-visible behavior:** none by default (`kind: oneshot` preserves today's behavior byte-for-byte). Operators who set `reviewer.kind: agentic` (Anthropic-shaped providers only, until a60) get on-demand file reading, no budget truncation, and discard-and-alert instead of default-approve on a missing verdict.
- **Acceptance:** `cargo test` passes; `openspec validate a58-agentic-reviewer --strict` passes. Tests: `kind: oneshot` is byte-identical to today; the agentic sandbox advertises `Read/Glob/Grep` + `submit_review` and NOT `Bash`/`Write`/`Edit`; a valid `submit_review` round-trips to `ReviewResult`; a non-enum verdict and a `should_request_revision`-without-`actionable_request` are rejected as correctable tool errors; a session with no submission discards the review and alerts (no auto-approve).
- **Dependencies:** stacks on **a56** (`agentic_run`, the `submit_*` framework, `record_submission`/`consume_submission`) and **a55** (`provider → CLI` strategy resolution). Independent of a57 (different role) and a53 (a58 does not touch the reusable-entry-point requirement; it preserves the `ReviewResult` shape that requirement and a53 build on). The default flip to `agentic` is deferred to **a60** (opencode), when the agentic path becomes provider-agnostic.

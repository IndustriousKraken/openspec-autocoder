## Why

To ensure the long-term maintainability of the orchestrator project itself, we need to establish a strict rule that documentation is a first-class citizen. Code changes generated by AI implementation agents must be accompanied by relevant updates to the README and architecture documents, preventing the documentation from becoming stale.

## What Changes

- Establish a new global project specification (`project-documentation`) that mandates documentation hygiene.
- Require that any changes to CLI commands, configurations, or architecture are reflected in the `README.md` and/or `docs/` folder.
- Ensure that implementation agents are forced (via OpenSpec rules) to include documentation updates in their PRs.

## Capabilities

### New Capabilities
- `project-documentation`: Defines the standards and requirements for maintaining living documentation within this repository.

### Modified Capabilities
- (none)

## Impact

This creates a foundational rule for all future changes. When an agent implements a new feature (like `multi-repo-manager`), the `openspec-apply-change` skill will read this documentation spec and automatically update the `README.md` as part of its implementation tasks, ensuring docs are never out of sync.
